# Инструкция по диагностике ошибок

## 1. Проверка консоли браузера

1. Откройте сайт в браузере
2. Нажмите `F12` (или `Cmd+Option+I` на Mac) для открытия DevTools
3. Перейдите на вкладку **Console**
4. Проверьте наличие ошибок (красные сообщения)
5. Скопируйте все ошибки и отправьте их

## 2. Проверка Network запросов

1. В DevTools перейдите на вкладку **Network**
2. Обновите страницу (`F5` или `Cmd+R`)
3. Проверьте, есть ли запросы со статусом 404 (не найдено) или 500 (ошибка сервера)
4. Особое внимание обратите на:
   - `.js` файлы
   - `.css` файлы
   - Компоненты React

## 3. Проверка DOM (Elements)

1. В DevTools перейдите на вкладку **Elements** (или **Inspector**)
2. Найдите элемент `<main>` или `<body>`
3. Проверьте, есть ли внутри контент:
   - `<header>` - должен быть виден
   - `<section>` - должен быть виден
   - Контент Hero секции

## 4. Проверка логов сервера

1. Откройте терминал, где запущен `npm run dev`
2. Проверьте, есть ли ошибки в логах
3. Скопируйте ошибки, если они есть

## 5. Тестовая страница

Откройте страницу `/debug` в браузере:
- Должна отображаться страница с инструкциями
- Если страница не загружается, значит проблема в Next.js

## 6. Проверка сборки

Запустите в терминале:
```bash
npm run build
```

Если сборка завершается с ошибками, скопируйте их.

## 7. Проверка компонентов

Попробуйте открыть страницы напрямую:
- `/debug` - тестовая страница
- `/technology` - страница технологии
- `/partners` - страница партнеров

Если эти страницы работают, а главная нет - проблема в компонентах главной страницы.

## 8. Частые проблемы

### Проблема: Белый/черный экран
- **Причина**: Ошибка JavaScript блокирует рендеринг
- **Решение**: Проверьте консоль браузера на ошибки

### Проблема: Загрузка без конца
- **Причина**: Компонент зависает при загрузке
- **Решение**: Проверьте динамические импорты в `app/page.tsx`

### Проблема: Только Header виден
- **Причина**: Контент скрыт или не рендерится
- **Решение**: Проверьте z-index и стили компонентов

## 9. Что отправить для диагностики

1. Скриншот консоли браузера с ошибками
2. Текст ошибок из консоли
3. Результат команды `npm run build`
4. Логи сервера разработки
